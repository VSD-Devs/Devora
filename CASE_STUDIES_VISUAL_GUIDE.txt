⚙️  HOW TO UPDATE A CASE STUDY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SCENARIO: You want to update "EnviroTech" case study data

STEP 1: Open the file
   👉 app/case-studies/[slug]/page.tsx

STEP 2: Find the caseStudies object
   👉 Look for: const caseStudies: Record<string, any> = {

STEP 3: Locate the entry you want to update
   👉 Find: "envirotech-plumbing": { ... }

STEP 4: Update the fields you need
   ✎ title: "New Title"
   ✎ challenge: "New challenge description"
   ✎ solution: "New solution description"
   ✎ results: { "Metric": "Value", ... }
   ✎ services: ["Service1", "Service2"]
   ✎ technologies: ["Tech1", "Tech2"]

STEP 5: Save the file
   The case study page updates automatically!

➕ HOW TO ADD A NEW CASE STUDY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SCENARIO: You want to add a 6th case study

STEP 1: Add to caseStudies object
   📄 File: app/case-studies/[slug]/page.tsx
   
   "your-new-slug": {
     title: "Your Company Name",
     subtitle: "Your tagline",
     challenge: "The problem...",
     solution: "How we solved it...",
     results: {
       "Metric 1": "Value",
       "Metric 2": "Value",
     },
     services: ["Service1", "Service2"],
     technologies: ["Tech1", "Tech2"],
   },

STEP 2: Add to projects array
   📄 File: components/our-work.tsx
   
   {
     name: "Your Company Name",
     slug: "your-new-slug",
     description: "Project description...",
     image: "/case-studies/your-new-slug.png",
     tags: ["Tag1", "Tag2"],
   },

STEP 3: Add the image
   📂 Add file: public/case-studies/your-new-slug.png

STEP 4: Done!
   The new case study is automatically available at:
   /case-studies/your-new-slug
